# Implementation Plan

## Tasks

- [x] 1. リストマーカースタイルの実装
- [x] 1.1 (P) 順序なしリストと順序付きリストの基本マーカースタイルを追加
  - `.prose ul`に`list-style-type: disc`を設定し黒丸マーカーを表示
  - `.prose ol`に`list-style-type: decimal`を設定し連番を表示
  - 既存の`.prose ul, .prose ol`スタイルブロックに`list-style-type`プロパティを追加
  - _Requirements: 1.1, 1.2, 3.1, 3.2_

- [x] 1.2 (P) ネストリストの階層別マーカースタイルを追加
  - `.prose ul ul`にcircleマーカーを設定（2階層目）
  - `.prose ul ul ul`にsquareマーカーを設定（3階層目）
  - ダークモードでのマーカー色はCSS変数による自動対応で追加作業不要
  - _Requirements: 1.3, 1.4, 3.1_

- [x] 2. 表スタイルの実装
- [x] 2.1 表の基本スタイルを追加
  - `.prose table`に幅100%、罫線の折りたたみ、マージンを設定
  - `.prose th, .prose td`にボーダー（`var(--border)`使用）とパディングを設定
  - `.prose th`にヘッダー背景色（`var(--muted)`使用）と太字を設定
  - _Requirements: 2.1, 2.2, 2.3, 2.4, 3.1, 3.2_

- [x] 2.2 表のダークモード対応と横スクロール機能を追加
  - CSS変数使用によりダークモード対応は自動的に行われることを確認
  - `.prose table`に`display: block`と`overflow-x: auto`を設定して横スクロール対応
  - 横幅が狭い画面でテーブルが横スクロール可能なことを確認
  - _Requirements: 2.5, 2.6_

- [ ] 3. 検証とテスト
- [ ] 3.1 リストと表のスタイル動作確認
  - 開発サーバーで記事ページを表示しリストマーカーが正常に表示されることを確認
  - 表を含むマークダウンで罫線、ヘッダー背景色、パディングが適用されていることを確認
  - ダークモード切り替えでスタイルが適切に変化することを確認
  - 既存のコードブロック、callout、link-cardのスタイルに影響がないことを確認
  - _Requirements: 1.1, 1.2, 1.3, 1.4, 2.1, 2.2, 2.3, 2.4, 2.5, 2.6, 3.3_

- [x] 3.2 ビルドとLintの実行
  - `pnpm lint`でCSSにエラーがないことを確認
  - `pnpm build`でビルドが正常に完了することを確認
  - _Requirements: 3.2_
