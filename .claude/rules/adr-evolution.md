# ADR 変更履歴の記録ルール

設計判断が途中で変更された場合、変更前の判断を削除せず経緯を保持する。

## 原則

1. **変更前判断を削除しない**: 「A → C に変更」形式で経緯を記録
2. **変更理由を明記**: なぜ当初案を撤回したかを具体的に書く
3. **抽象的な表現を禁止**: 「より良い」「改善」ではなく、具体的な問題点を記述

## フォーマット

```markdown
### 採用: A → C に変更。{変更後の名称}

当初は A（{当初案の概要}）を採用したが、{きっかけ}で再検討し C に変更。

**当初案 A の問題点:**
- {具体的な問題点1}
- {具体的な問題点2}

**採用: C. {変更後の名称}**
- {採用理由1}
- {採用理由2}
```

## 使用タイミング

- レビュー・議論で設計判断が変更されたとき
- 実装中に当初の設計では問題があると判明したとき
- 判断が変更されていない場合は通常フォーマットで十分
