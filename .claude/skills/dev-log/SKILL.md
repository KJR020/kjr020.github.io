# 実装ログ記録スキル

実装作業の振り返りと学びを日報形式で記録するスキル。

## 使用タイミング

- 機能実装が完了したとき
- バグ修正が完了したとき
- 学びがあったとき

## 出力先

`.claude/personal_docs/dev-log/YYYY-MM-DD-{feature-name}.md`

## テンプレート

```markdown
# {日付} - {機能名}

## 概要
{1-2文で何をしたか}

## 問題
{解決しようとした課題}

## 解決策
{採用したアプローチとその理由}

## 学び
{今回得られた知見・気づき}

## ハマりポイント
{詰まった箇所と解決方法}

## 次回への申し送り
{関連する今後のタスクや注意点}
```

## 書き方のポイント

- 主観で書く（「〜と思った」「〜が面白かった」など）
- 技術的な詳細より「なぜその判断をしたか」を重視
- 失敗や迷いも正直に記録
- 短くても良いので継続できる粒度で
